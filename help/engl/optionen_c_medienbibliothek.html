File management

Configure the media library.

Directories
Nemp can scan your music directories for new files on every start to keep your media library up-to-date.
  • Automatically monitor new directories: Whenever you add a directory to your library (e.g. by dropping it from the Windows explorer), this directory will be added to this list.
  • Show query dialog when adding new directories will show a dialog whether Nemp should monitor this directory or not.

Filetypes for the library
Nemp is able to play a lot of filetypes, but some of them can not be analyzed for the media library. Therefore you can choose the filetypes you want to add to the library. 
Tip: Nemp can handle very well the following types: mp3, ogg, flac, m4, ape, mpc, ofr, wv and tta. Windows Media Audio (wma) is not fully supported.

File type registration

Register Nemp as the default player for audio files and specify what to do with new files. "Enqueue" will add the new file to the last playlist, otherwise the current playlist is cleared and the new file will be the only one in the playlist.

Meta data (e.g. ID3-Tags)

Nemp uses meta tags to store some information directly into the files, e.g. the user rating and the play counter. This will change the files quite often, which may collide with your backup strategy.
Note 1: This setting is OPT-IN, and one of the Nemp wizard settings.
Note 2: The file is definitely changed, when you use the "Properties" dialog to change the meta data.

Cover

While adding files to your library, Nemp will search for matching cover art for the audio files. Nemp will use the embedded cover art in the files and image files on your hard disk, if there are any. You can specify, where Nemp should scan for a matching cover art.

Search options (extended settings)

As almost everytime in computer science, you have the choice to do something fast (using a lot of memory), or to do it a bit slower (with less memory).
The Nemp quicksearch feature gives you the choice to adjust the memory usage.

Recommended is the accelerated search, which will use approximately 1mb for 4000 files, maybe some more if you include filenames (recommended) and comments (often not useful).

Note: If you exclude filenames, the files should contain proper meta data if you want to find them through the quicksearch.

Quicksearch options
Nemp can give you proper search results in (almost) real time, "while you type". If you have many files in your library (> 10.000 files), accelerated search is strongly recommended for this feature.
In addition, Nemp can do a fuzzy search to handle some typos (in your search query as well as in the media library). This will take a little bit more time, so the fuzzy search is only done after hitting the enter key (default setting). If your PC is pretty fast, an on the fly fuzzy search may work as well.

Tip: Fuzzy search is also useful for artists like Pink (or P!nk?) and Alanis Morissette (how many r, s, t are correct?).

Unicode (extended settings)

Some mp3 files doesn't support unicode very well, and use corrupt char encoding. Nemp can try to guess the correct one by analyzing the filename: If a file is named with a lot of greek letters, it can be assumed that the information within the ID3-Tags is also in greek.
Tip: If you dont know what I'm talking about: Just ignore this setting. ;-)
