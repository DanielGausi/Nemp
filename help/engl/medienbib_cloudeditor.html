The tag cloud editor

If you use last.fm for getting additional tags for the tracks in your library, you may get some useless tags, or identical tags with different spellings.

With the tag cloud editor you can clean up this mess a little bit.

Existing tags
A list of all (really all) tags in your library. From popular ones as "Singer-Songwriter" to very exotic ones like "songs, die so gut sind das ich meiner oma ihr klein haeuschen zwar nicht verkaufen aber zumindest dafuer beleihen wuerde". (The last one actually exists, and translates to "songs which are that good, that I won't sell my grandma's cottage for it, but maybe hypothecate it").
You can rename these tags, merge or delete them. Just select one ore more tags and click the appropriate button.

The setting "Hide Auto-Tags" will hide tags, that Nemp created on its own (like Artist, Album, Year, Genre, Decade). You cannot change these tags completely. However you can change these tags for files, where Nemp did not automatically created them.

The setting "Remember merged/renamed/deleted tags" will save your input, so that following queries to last.fm will be automatically corrected.
Example: You merge the two tags "Female Vocalist" and "Female Vocalists" to "Female Vocalists". If a later query to last.fm replies "Female Vocalist", this tag will automatically be changed to "Female Vocalists".


Merged/renamed tags
A list of all modified tags. This is the list Nemp uses to adapt the last.fm replies.


Deleted Tags
Tags, you don't want to see in your tag cloud. If last.fm reply contains one of these tags, it is just ignored.


Update files now
After renaming/merging/deleting tags, you should change the tags in your mp3-files as well. This may take some time - probably a little bit longer than reading the information while scanning your harddrive for new files.
