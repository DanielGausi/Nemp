Der Nemp Webserver

Der Nemp Webserver bietet die Möglichkeit, von einem Browser aus über ein Netzwerk auf den Player zuzugreifen.

Webserver aktivieren
Sie aktivieren den Webserver durch Klick auf den Button "Server aktivieren". Im Logfenster sehen Sie dann, wer darauf zugreift und was der da so macht.
Ist der Webserver aktiviert, wird im Hauptfenster ein Symbol eingeblendet.

Sie können mit dem Webserver 
  • Den aktuellen Titel des Players im Browser sehen
  • Den aktuellen Titel herunterladen (z.B. auf das Handy, mit dem Sie Nemp betrachten)
  • Den Player steuern
  • Die Playliste anzeigen lassen und einzelne Titel herunterladen
  • In der Medienbibliothek suchen und Titel herunterladen
  • Titel zur Playlist hinzufügen
Sie können für den Zugriff auf den Webserver einen Nutzernamen und Passwort eingeben, um Unbefugten den Zugriff zu verweigern.


Zugriff aus dem lokalen Netz
Geben Sie in ihrem Browser die "Eigene IP (LAN)" ein. Wenn Sie den Webserver auf dem Rechner testen wollen, auf dem auch gerade Nemp läuft, verwenden Sie die IP 127.0.0.1.
Die IP-Adresse müssen Sie da eingeben, wo Sie sonst z.B. www.gausi.de aufrufen - in der Adresszeile des Browsers. Das Suchfeld von Google ist nicht die Adresszeile.


Zugriff von außerhalb
Wenn Sie Zugriff von außerhalb erlauben wollen, müssen Sie zunächst Ihre Internet-IP ermitteln lassen. Das geschieht über ein kleines PHP-Script auf meiner Seite www.gausi.de, das durch Klick auf den Button "IP-Adresse ermitteln" aufgerufen wird. Diese IP müssen Sie Ihren Freunden zusammen mit Nutzernamen und Passwort mitteilen.

Hinweis: Der Aufruf dieses Scripts von Ihrer Adresse wird zwar (wie jeder Seitenaufruf fast überall im internet) von dem Apache auf dem Server geloggt. Sie wird aber nicht weiter verwendet oder gar an Dritte weitergegeben. Natürlich können Sie dafür auch andere Dienste im Netz zur Ermittlung Ihrer IP-Adresse verwenden.

Zusätzlich müssen Sie ggf. in der Konfiguration Ihres Routers den Port 80 auf Ihre lokale IP-Adresse weiterleiten. Dadurch ist sichergestellt, dass die HTTP-Anfragen aus "dem Internet" bei Nemp auch ankommen und nicht am Router hängenbleiben.


Warnung!

Wenn Sie den Zugriff von außerhalb erlauben und kein Passwort setzen, dann kann jeder auf Ihre gesamte Musiksammlung zugreifen. Sie haben damit praktisch eine eigene "Internetseite" erstellt und bieten auf dieser Seite hunderte oder gar tausende möglicherweise urheberrechtlich geschützte Lieder zum Download an.

Das ist eine sehr schlechte Idee und kann sehr kostenintensive Folgen haben!


Sie können den Zugriff daher generell auf das lokale Netz beschränken. HTTP-Anfragen von außerhalb werden dann mit einer Fehlermeldung beantwortet.
