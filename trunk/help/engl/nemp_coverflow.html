The Nemp coverflow



The Nemp coverflow may work differently from other players. This is how the coverflow works in Nemp:
  1. Every file is searched for embedded cover art. If a file contains embedded cover art, this one is used as a cover for this file.
  2. If there is no embedded cover art (probably the case for more than 90%), Nemp searches for image files "nearby" (see below) the audio file.
  3. If more than one image file is found, Nemp tries to get the "front cover". For this, the filenames are checked: For example, a file containing "front" or "folder" seems to be a good choice.
  4. All audio files, where Nemp has found the "same cover" (determined by the MD5-checksum of the image data), are grouped together as "one album". 
  5. If no images can be found, all files in one directory are grouped together as "one album".
"Nearby" depends on the settings (File management -> Cover). Default setting is
  • Search in the directory of the audio file itself
  • Search in a subdirectory named "cover"
  • Search in the parent directory (only if the parent directory contains only a few directories) 
  • Search in a subdirectory of the parent directory named "cover" (only if the parent directory contains only a few directories)

Sorting the Coverflow
You can change the sort order in the coverflow. Just use the context menu and choose "Browse by -> ...".
Note that the artist of a sampler is just "Various Artists".


Download missing covers
If no cover art can be found, Nemp can try to find a proper image online. For this the API of last.fm is used, which is also used for scrobbling. You do not need a last.fm account for downloading cover art.

If the last.fm API was used to display a cover, Nemp will add a small icon to the cover to give you some feedback whether the download was successful (or not):

The cover has been found and downloaded. A copy of this image is now stored in the directory of the audio file (named "front (NempAutoCover).jpg", sometimes *.png) .
Nemp was not able to find a matching cover. This happens for uncommon albums and (unfortunately) quite often for samplers
The query to last.fm was skipped, as the same query failed earlier. Nemp will try it again a few times later (next day/week). If the query is failing all the time, Nemp will block further queries for that album completely (until the cover cache is cleared, Settings -> File management -> Cover).
No connection to the last.fm webservice. You should check your internet or firewall settings.
Used in the player, if Nemp was not able to find proper "Album information" for the current track. If Nemp is unsure about the album (e.g. the file is not in contained in the media library), the last.fm query is skipped.
